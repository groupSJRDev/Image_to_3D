I can see exactly what's happening. The model is computing landmark positions and rotations independently, so arms end up detached and misrotated. The fix: replace the freeform landmark math with a joint chain system where each limb segment starts exactly where the previous one ended.

Root Cause Diagnosis
Problem in render	Cause
Arms floating away from torso	Model estimated arm position independently from shoulder position
Arms at wrong angle	atan2 computation error (LLMs are bad at trig)
Disconnected joints	No enforcement that elbow = end of upper arm
Duplicate/mirrored figure	Possibly multi-object scene logic triggered
The Fix: Joint Chain System

Instead of "estimate all landmarks, then compute positions/rotations," use: "Start at root joint → pick direction → compute center and endpoint mechanically → chain to next segment"

This guarantees connectivity.

═══════════════════════════════════════════════

Before generating ANY geometry, perform a structured analysis:

1. IDENTIFY
 - What exactly is this object (or scene of objects)?
   Be specific (not "chair" but "mid-century modern office
   swivel chair with 5-star base")
 - For figures: describe the POSE precisely — which limbs are
   where, what direction they face, what angles they make.
 - What is its primary function? This informs structural logic.
 - If MULTIPLE objects: list each one separately.

2. MEASURE (estimate real-world dimensions)
 - Overall bounding box per object: width (X) × height (Y) × depth (Z)
 - Use known reference sizes to calibrate:
   Human hand ~0.18m, doorknob height ~1.0m, seat height ~0.45m,
   table height ~0.75m, soda can ~0.12m tall, car tire ~0.65m dia,
   burger 0.10-0.12m diameter, pint glass 0.07m × 0.18m tall
 - Human body: head ~0.23m tall, shoulder width ~0.45m,
   full height ~1.75m, torso length ~0.55m,
   upper arm ~0.30m, forearm ~0.26m, hand ~0.19m,
   thigh ~0.45m, shin ~0.42m, foot ~0.26m
 - If analyzing an image: note the camera angle and estimate
   what foreshortening or perspective may distort proportions.

3. DECOMPOSE into structural components
 List every distinct part you can identify:
 - Name each part
 - Estimate its dimensions (width, height, depth or radius)
 - Describe its position relative to other parts
 - Note which geometry type best represents it
 - Flag any parts that are HIDDEN or OCCLUDED but must exist
   (e.g., back legs hidden behind front legs, underside structure)

3b. SILHOUETTE TRACE (for curved objects)
  If the object or any component has smooth curves:
  - Identify the axis of symmetry in the image
  - Trace the outer edge from bottom to top
  - Note every point where the contour changes direction,
    curvature, or rate of curvature change
  - List these as approximate (x, y) values in meters
  - This trace becomes your source of truth for profilePoints
  - COUNT your trace points. If you have fewer than 20 for
    a curved object, you are undersampling.

3c. LAYER STACK MAP (for stacked/layered objects)
  If the object has layers stacked vertically (burgers,
  sandwiches, cakes, geological strata, stacked books):
  - List every layer from BOTTOM to TOP

    • name
    • thickness (height in meters)
    • radius or width/depth
    • whether it OVERHANGS the layer below
    • whether it COMPRESSES into the layer below
  - Compute running Y positions right here in the analysis:
    Layer 1: base=0.000, center=0.xxx, top=0.xxx
    Layer 2: base=0.xxx, center=0.xxx, top=0.xxx
    ...
  - These computed Y values become your ONLY source of truth
    for position.y in the geometry. Do not recalculate later.

  - DO NOT estimate elbow, wrist, knee, ankle positions here.

CONTOUR TRACING PROTOCOL (for curved/organic shapes)
═══════════════════════════════════════════════

When the object has smooth curves (bottles, vases, cups, buns,
rounded forms), you MUST trace the actual contour — not
approximate it from memory.

SILHOUETTE SAMPLING METHOD:
1. Mentally place the object's silhouette on a grid
2. Divide the total height into slices at ~2-3% intervals
 (a 0.3m bottle → sample every ~0.006-0.009m = 30-50 points)
3. At EACH slice, measure the horizontal distance from center
 axis to the outer edge — this is your x value
4. Add EXTRA points wherever curvature changes sharply:
 - Transitions (body→shoulder, shoulder→neck, neck→lip)
 - Inflection points (convex → concave)
 - Flat runs need fewer points, tight curves need more
5. Read measurements from THE IMAGE, not from what you think
 the object "should" look like

MINIMUM POINT DENSITY:
- Simple curves (gentle taper): 1 point per 1cm of real height
- Complex curves (shoulder, lip): 3-5 points per 1cm
- Straight sections: 1 point at start + 1 at end is sufficient
- Total for a typical bottle/vase: 30-60 profile points

COMMON FAILURE MODES TO AVOID:
✗ "Generic bottle" — 6 points making a vague bottle shape
✗ Symmetric shoulder — most real objects have different
  curvature entering vs exiting a transition
✗ Perfectly straight sections — real objects have subtle taper
✗ Missing the lip/rim detail — often the most recognizable feature
✗ Skipping the base profile — objects often have a foot/edge detail

STACKING PROTOCOL (for layered objects)
═══════════════════════════════════════════════

For ANY object composed of stacked layers (burgers, sandwiches,
cakes, tiered structures), follow this procedure.

PROCEDURE:
runningY = 0.0  (or surface height if on a table/plate)

For each layer, bottom to top:
  layerBase   = runningY
  layerCenter = runningY + (layerHeight / 2)
  layerTop    = runningY + layerHeight
  → position.y = layerCenter
  → runningY   = layerTop

NESTING ADJUSTMENTS:
Some layers compress into soft layers below them:
- A heavy patty sinks ~1-2mm into a soft bun
- Subtract the nesting depth from runningY before placing:
  runningY -= nestingDepth

OVERHANG:
Some layers extend horizontally beyond their neighbors:
- Lettuce extends 20-40% beyond bun radius
- Melted cheese drapes 10-20% beyond patty
- These layers have LARGER radius/width but same Y stacking

VERIFY: After computing all layers, confirm:
totalStackHeight = final runningY
Does this match your Phase 1 overall height estimate?
If not, adjust layer thicknesses proportionally.

ARTICULATED FIGURE PROTOCOL — JOINT CHAIN SYSTEM
═══════════════════════════════════════════════

For human figures, animals, robots, or any articulated character,
use the JOINT CHAIN system. This guarantees limbs connect properly.

DO NOT independently estimate positions for elbows, wrists, knees.

║  DIRECTION → ROTATION & OFFSET TABLE     ║

║   source of truth for limb placement)    ║

A cylinder's default axis is Y (pointing UP in Three.js).
To point it in a direction, apply rotation.z.
To compute position, offset from the START JOINT.

For INTERMEDIATE angles, interpolate between the two nearest
entries. E.g., 45° is halfway between 30° and 60°:
rot.z = -0.785
center offset = (+L*0.35, +L*0.35, 0)
endpoint offset = (+L*0.71, +L*0.71, 0)

For DEPTH (limb going toward/away from camera):
Apply rotation.x instead of (or in addition to) rotation.z.
Toward viewer: rotation.x = positive
Away from viewer: rotation.x = negative
Use same offset logic but swap Z for the depth component.

GEOMETRY SELECTION RULE:
- Smooth curved silhouette → LATHE with dense profile
- Irregular/ruffled flat shape → EXTRUDE with wavy path
- Known basic shape, no curvature detail → PRIMITIVE
- Flat decorative 2D shape → EXTRUDE
- Curved elongated path → TUBE
- Articulated figure limb → CYLINDER (tapered)
- Figure head → SPHERE (egg-scaled) + mandatory features
- Figure hand/foot → BOX or detailed sub-parts
- Nose on a face → CONE with radialSegments: 4 (pyramid)

NEVER approximate a visible curve with a primitive.

═══════════════════════════════════════════════
MULTI-OBJECT SCENES
═══════════════════════════════════════════════

When the image contains multiple distinct objects:
1. Identify each separate object and its ground footprint
2. Estimate horizontal spacing between object centers
3. Build each object independently with correct X/Z offsets
4. All objects share the same ground plane (Y=0)
5. Left object → negative X, Right → positive X
6. Label parts with object prefix: "glass-body", "burger-top-bun"

═══════════════════════════════════════════════
DESIGN STRATEGY
═══════════════════════════════════════════════

Think like an architect — decompose into structural components:
Table    → 1 box (top) + 4 box (legs)
Chair    → seat slab + 4 legs + 2 back posts + back slat
Car      → body box + cabin box + 4 wheel cylinders + axle boxes
Lamp     → base cylinder + pole cylinder + shade cone
Burger   → bottom bun (lathe) + lettuce (extrude wavy) +
           patty (cylinder) + tomato (thin cylinders) +
           onion (torus) + cheese (thin box) + top bun (lathe)
Glass    → glass body (lathe) + liquid cylinder + ice (boxes)
Person   → JOINT CHAIN: torso boxes + neck cylinder +
           egg head + facial features (eyes, nose, mouth, ears) +
           hair + arm cylinders + hand boxes + leg cylinders +
           foot boxes. Use the chain table for all positions.

Use enough parts for RECOGNIZABLE wireframe silhouette.
Typical objects: 5–30 parts.
Articulated figures: 20–45 parts (including 7-8 head parts).

═══════════════════════════════════════════════
CONSTRAINTS
═══════════════════════════════════════════════

- Y-up coordinate system
- Units: meters
Human full height ~1.75m, head ~0.23m tall
- Ground plane: lowest surface at Y=0
- Each part in WORLD SPACE
- Rotations in radians
- Colors: hex strings, distinct per logical group
Figures: head, torso, left arm, right arm, left leg, right leg, hands
- CRITICAL FOR FIGURES: Every limb position and rotation MUST come
from the Joint Chain computation. Do NOT estimate limb positions
independently. Chain from root joints outward.

POSITION CALCULATION RULE:
Position = CENTER of geometry, not its base.

FOR STACKED LAYERS — copy Y from Phase 1 Layer Stack Map.

FOR FIGURE LIMBS — use center position computed by the
Joint Chain Table. The center = start_joint + center_offset.
DO NOT estimate positions independently.

FOR HEAD FEATURES — use headCenter (from chain) + offsets
from Face Feature Map.

FOR EXTRUDED FLAT SHAPES — rotate x: -π/2, position.y = layer center.

VALIDATION BEFORE OUTPUT:
□ Do repeated parts share identical dimensions?
□ Does total assembled height match Phase 1 estimate?
□ Do proportional ratios hold?
□ Is the object grounded (Y=0)?
□ For lathe: 20+ points, dense at curves?
□ For stacked objects: no gaps/overlaps?
□ For multi-object scenes: correct spacing?
□ FOR FIGURES — CONNECTIVITY CHECK (CRITICAL):
  □ Does upper arm START at shoulder position?
  □ Does upper arm END at elbow = shoulder + endpoint_offset?
  □ Does forearm START at that exact elbow position?
  □ Does forearm END at wrist = elbow + endpoint_offset?
  □ Does hand START at that exact wrist position?
  □ Are LEFT and RIGHT limbs same dimensions, different rotations?
  □ Do shoulders branch from neckBase + horizontal offset?
  □ Does neck sit on top of upper torso?
  □ Does head sit on top of neck?
  □ ALL positions computed by chain table, NONE estimated freely?
□ FOR HUMAN HEADS — mandatory parts:
  □ head-cranium (egg sphere)
  □ head-eye-left (sphere)
  □ head-eye-right (sphere)
  □ head-nose (cone, radialSegments:4)
  □ head-mouth (box)
  □ head-ear-left (box)
  □ head-ear-right (box)
  □ head-hair (unless clearly bald)
□ Eyes identical and symmetric?
□ Ears identical and symmetric?
□ Nose points outward from face?
□ Hair covers top of cranium?

What this fixes:

Complex multi-step trig	"compute atan2, then midpoint, then..."	"Look at image → pick clock direction → look up in table"
Inconsistent results	Different approaches each generation	Mechanical procedure with worked example to pattern-match

The worked example is arguably the most impactful addition — it shows the model exactly what the numbers should look like for a real pose, giving it a concrete pattern to follow rather than abstract formulas.
